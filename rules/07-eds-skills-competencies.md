# EDS Skills & Competencies Guide

## Overview
This document outlines the required skills, knowledge areas, and competencies for working effectively with Adobe Edge Delivery Services (EDS). Use this for training, hiring, skill assessment, and career development.

---

## Skill Levels

### 🟢 Level 1: Beginner
- Basic understanding of concepts
- Can perform simple tasks with guidance
- Learning and practicing

### 🟡 Level 2: Intermediate
- Good understanding of concepts
- Can work independently on standard tasks
- Can troubleshoot common issues

### 🔴 Level 3: Advanced
- Expert understanding
- Can handle complex scenarios
- Can mentor others and create best practices

---

## Core Technical Skills

### 1. HTML & Semantic Markup

#### 🟢 Beginner
- [ ] Understand basic HTML structure (doctype, html, head, body)
- [ ] Know common elements (div, span, p, h1-h6, a, img)
- [ ] Understand attributes (class, id, src, href, alt)
- [ ] Create simple page structures

**Learning Resources**:
- [MDN HTML Basics](https://developer.mozilla.org/en-US/docs/Learn/HTML)
- [HTML5 Semantic Elements](https://www.w3schools.com/html/html5_semantic_elements.asp)

#### 🟡 Intermediate
- [ ] Use semantic HTML5 elements (article, section, nav, aside, header, footer)
- [ ] Understand document outline and heading hierarchy
- [ ] Implement accessibility attributes (ARIA labels, roles)
- [ ] Create forms with proper validation
- [ ] Optimize HTML for performance (lazy loading, preloading)

**Learning Resources**:
- [MDN Semantic HTML](https://developer.mozilla.org/en-US/docs/Glossary/Semantics)
- [Web Accessibility Initiative](https://www.w3.org/WAI/)

#### 🔴 Advanced
- [ ] Design accessible, SEO-optimized HTML structures
- [ ] Implement complex ARIA patterns
- [ ] Optimize HTML for Core Web Vitals
- [ ] Create reusable HTML patterns
- [ ] Review and refactor HTML for best practices

#### 🔴 Advanced

## EDS-Specific Skills

### 6. EDS Fundamentals

#### 🟢 Beginner
- [ ] Understand EDS/Franklin architecture
- [ ] Know the concept of blocks
- [ ] Understand document-based authoring
- [ ] Know how content is served
- [ ] Understand the relationship between Google Docs and web pages

**Key Concepts**:
- Blocks are the building units
- Content comes from Google Docs/SharePoint
- No traditional CMS
- Fast, performant by default

**Learning Resources**:
- [EDS Documentation](https://www.aem.live/docs/)
- [Franklin GitHub](https://github.com/adobe/franklin)

#### 🟡 Intermediate
- [ ] Create and modify blocks
- [ ] Understand block decoration
- [ ] Work with EDS utilities (aem.js, scripts.js)
- [ ] Implement responsive blocks
- [ ] Use EDS image optimization
- [ ] Work with metadata
- [ ] Understand lazy loading patterns

**Key Skills**:
- Block decoration function pattern
- Using `createOptimizedPicture`
- Loading CSS and JS efficiently
- Working with the DOM

#### 🔴 Advanced
- [ ] Design block architectures
- [ ] Implement complex blocks with interactions
- [ ] Optimize performance (Core Web Vitals)
- [ ] Create reusable patterns
- [ ] Implement advanced lazy loading
- [ ] Debug EDS-specific issues
- [ ] Contribute to EDS core utilities

## Learning Path

### Path 1: Frontend Developer → EDS Developer

**Phase 1: Foundations (2-4 weeks)**
- HTML, CSS, JavaScript fundamentals
- Git basics
- Command line basics
- Browser DevTools

**Phase 2: EDS Basics (2-3 weeks)**
- EDS architecture understanding
- Block creation and modification
- Local development setup
- Basic blocks (hero, cards, columns)

**Phase 3: Intermediate (4-6 weeks)**
- Complex block creation
- Component patterns
- Brand theming
- Performance optimization
- Accessibility basics

**Phase 4: Advanced (Ongoing)**
- Component architecture
- Multi-brand systems
- Advanced performance
- Mentoring others

### Path 2: Backend Developer → EDS Developer

**Phase 1: Frontend Fundamentals (4-6 weeks)**
- Modern HTML/CSS
- JavaScript (ES6+)
- DOM manipulation
- Async programming

**Phase 2: EDS Specifics (3-4 weeks)**
- EDS architecture
- Block patterns
- Content-first approach
- Static site concepts

**Phase 3: Integration (4-6 weeks)**
- API integration
- Form handling
- Data fetching
- Advanced interactions
